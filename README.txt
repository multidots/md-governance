Easily manage block visibility and configuration in the Gutenberg editor based on user roles with MD Governance.
  
== Description ==  
MD Governance empowers WordPress administrators to control access and settings of Gutenberg blocks for different user roles. This plugin is ideal for teams, agencies, or site managers who want fine-grained control over the editing experience in the block editor.

**Key Features**:  
<ul>
<li><strong>Enable/Disable Blocks:</strong> Choose which blocks (core or custom) are available in the editor per user role.</li>
<li><strong>Settings Visibility:</strong> Control access to block settings for selected roles to avoid misconfigurations.</li>
<li><strong>Role-Based Customization:</strong> Tailor the Gutenberg experience for editors, authors, contributors, and more.</li>
<li><strong>Supports All Blocks:</strong> Works with default WordPress blocks and custom third-party blocks.</li>
</ul>

== Installation ==  
1. Upload the `md-governance` folder to the `/wp-content/plugins/` directory.  
2. Activate the plugin through the 'Plugins' menu in WordPress.  
3. Navigate to **Settings > MD Governance** to manage block visibility and role-based access.

== Frequently Asked Questions ==  

= What blocks can I manage with this plugin? =  
You can control access to all default WordPress blocks and any additional blocks added via other plugins.

= Can I restrict block settings separately from block visibility? =  
Yes, the plugin allows you to hide block settings while keeping the block visible in the editor.

= Will this work with custom user roles? =  
Absolutely! MD Governance supports all user roles, including custom ones.

= Does this affect the front-end rendering of blocks? =  
No. MD Governance only controls visibility and settings within the Gutenberg editor, not the front-end output.

= Is this plugin compatible with Full Site Editing (FSE)? =  
Currently, the plugin is focused on the post and page editors, with future plans for FSE support.
